# 框架与技术栈

---

## 一、主流框架对比

| 框架 | 理念 | 学习曲线 | 核心优势 | 核心劣势 |
|---|---|---|---|---|
| **OpenAI Agents SDK** | 极简原语 | ⭐ 最低 | 零学习成本，内置追踪 | 供应商锁定 |
| **CrewAI** | 角色扮演团队 | ⭐⭐ 低 | `kickoff()` 一键跑 | 单线程，高并发慢 |
| **LangGraph** | 图+状态机 | ⭐⭐⭐⭐ 高 | 检查点、回滚、人在回路 | 前期建模成本高 |
| **AutoGen** | 对话驱动 | ⭐⭐⭐ 中 | 灵活多 Agent 对话 | 对话可能失控 |

### 场景速查

| 你的情况 | 推荐 | 原因 |
|---|---|---|
| 1 天出 Demo 给老板看 | CrewAI | 最快路径 |
| OpenAI 全家桶 + 快速投产 | OpenAI Agents SDK | 原生集成 |
| 金融/医疗等合规场景 | LangGraph | 审计追踪、状态回滚 |
| 多 Agent 研究实验 | AutoGen | 灵活对话模式 |
| 复杂 10+ 步骤工作流 | LangGraph | 条件分支、断点恢复 |

### 学习顺序

```
OpenAI Agents SDK（1天）→ CrewAI（1-2天）→ LangGraph（5天，重点掌握）
```

> 掌握 LangGraph 基本能吃透生产级 Agent 的状态管理。

## 二、核心技术栈

```
语言：      Python（首选，>78% Agent 生态）+ TypeScript（前端/全栈 Agent）
API：       OpenAI API / Anthropic API
框架：      OpenAI Agents SDK → LangGraph（主力）
向量数据库： Chroma（入门）→ Milvus（生产）
RAG：       LlamaIndex / LangChain
评估：      Ragas / LangSmith / Braintrust
可观测性：  Langfuse / OpenTelemetry
部署：      FastAPI + Docker + K8s
协议：      MCP（工具接入）/ A2A（Agent 间通信）
```

---

## 三、为什么 LangGraph 是重点

LangGraph 的图+状态机模型对应生产级 Agent 的核心需求：

| 需求 | LangGraph 怎么解决 |
|---|---|
| 执行到第 7 步挂了 | Checkpoint 断点恢复 |
| 需要人工审批 | Human-in-the-loop 节点 |
| 不同条件下走不同分支 | 条件边（Conditional Edge） |
| 出错了要回退 | 错误边（Error Edge）+ 状态回滚 |
| 同时做多件事 | 并行分支 |
| 需要审计"Agent 为什么这么做" | 每个节点状态可追溯 |

---

## 四、不推荐（现阶段）

- **LangChain 高级抽象**：封装太厚，不利于理解本质
- **Dify / Coze**：低代码平台，不教你工程能力
- **从零手写一切**：学习时可以，生产不要重复造轮子
