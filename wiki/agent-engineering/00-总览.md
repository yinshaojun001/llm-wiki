---

## Agent 是什么

Agent 本质 = LLM + Memory + Planning + Tool Use

```
用户输入 → Agent（思考 → 调用工具 → 观察结果 → 再思考）→ 产出结果
```

不是一次性问答，而是自主的多步推理循环。

## 每周投入

- **时间**：10-12h/周（工作日 3 晚 × 1.5h + 周末一天 4h + 另一天自由）
- **比例**：60% 写代码 + 30% 看文档/视频 + 10% 总结笔记
- **每个产出**：可运行的代码仓库 + 一篇 Obsidian 笔记
- **月度复盘**：对比月初目标，调整下月重点

---

## 学习原则

1. **学机制，不学名字**——模型名和框架名会过时，机制持续复利
2. **构建产出物，而非囤积观点**——每阶段交付可演示的代码/评估报告/架构图
3. **评估一切**——用 evals、traces、失败分类做决策，不靠感觉
4. **先手写，再框架**——理解本质后再用框架提效
5. **你的 Java 经验全部复用**——可靠性、监控、发布、异常处理，Agent 时代完全需要
