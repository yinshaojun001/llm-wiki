# 第四阶段：落地与生产化

> 第 37-52 周 | 目标：掌握从 POC 到 Production 的完整链路，建立求职 Portfolio。

---

## 第 37-38 周：生产架构设计

### 怎么学

| 天 | 内容 |
|---|---|
| 1-2 | 模型路由——多模型切换架构（大模型推理、小模型分类），不绑死一个供应商 |
| 3-4 | 服务化部署——FastAPI 封装 Agent，Docker 化，健康检查 + 优雅关闭 |
| 5-6 | 自动扩缩容——Agent 任务入队，Worker 按需扩容（K8s HPA 或 Serverless） |
| 7 | 画出生产架构图（draw.io 或 Excalidraw） |

### 产出

一张生产级 Agent 服务架构图 + Docker Compose 一键启动方案。

---

## 第 39-40 周：渐进式发布

```
Sandbox（内部测试）
  → Shadow Mode（静默跑在真实流量旁，只收集 trace）
    → Canary（1% 用户）
      → 5% → 25% → 100%（每个阶段所有质量门禁都绿才推进）
        → 回滚（每个阶段都有可执行回滚方案）
```

### Java 类比

和金丝雀发布/灰度发布完全一样——就是你每天在做的发布流程。

### 产出

一份可执行的渐进发布方案文档 + 回滚操作手册。

---

## 第 41-42 周：成本优化

| 天 | 内容 |
|---|---|
| 1-2 | Token 预算——按任务复杂度分级设置 token 上限 |
| 3-4 | SLM 替代——简单任务（分类、摘要）用小模型，核心推理才用大模型 |
| 5-6 | Prompt Caching——Anthropic/OpenAI cache 机制，重复 system prompt 不再重复计费 |
| 7 | 成本实验——同一任务用不同策略，对比成本差异（目标降 50%+） |

### 产出

成本优化方案 + 降本 50%+ 的实验数据。

---

## 第 43-44 周：安全合规

| 天 | 内容 |
|---|---|
| 1-2 | PII 脱敏——手机号/身份证/银行卡号自动识别和脱敏 |
| 3-4 | 审计日志——谁、什么时候、触发 Agent 做了什么、结果如何，不可篡改 |
| 5-6 | 权限最小化——Agent 只有当前任务所需最小权限，每次单独申请短期凭证 |
| 7 | Prompt 防火墙——部署注入检测模型，拦截恶意 prompt |

### 产出

安全合规清单 + 审计日志实现 + PII 脱敏模块。

---

## 第 45-46 周：多 Agent 生产系统

### 做什么

把之前的多 Agent Demo 升级为生产级——加入 session 管理、可观测性、渐进发布、熔断。

### 产出

一个企业级多 Agent 系统 Demo。

---

## 第 47-48 周：综合 Portfolio 项目

### 做什么

自选一个真实业务场景，从 0 到生产交付完整 Agent 系统。**建议选你熟悉的 58 业务场景**。

### 要求

- [ ] 完整评估体系（Golden Test Set + LLM-as-Judge）
- [ ] 全链路可观测性（Langfuse）
- [ ] 安全护栏（四层防护）
- [ ] 成本分析报告
- [ ] 系统架构图
- [ ] 能 Demo 给面试官看

### 产出

Portfolio 项目（GitHub 开源仓库 + 技术博客 + Demo 视频）。
