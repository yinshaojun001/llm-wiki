## 1. 装饰器

```python
import time
from functools import wraps

# 基本装饰器
def timer(func):
    @wraps(func)  # 保留原函数的元信息
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start:.2f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    return "done"

slow_function()  # 打印: slow_function took 1.00s，返回 "done"

# 带参数的装饰器
def retry(max_attempts: int = 3):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_attempts - 1:
                        raise
                    print(f"Attempt {attempt + 1} failed: {e}")
        return wrapper
    return decorator

@retry(max_attempts=3)
def unstable_api():
    ...
```

**Java 类比**：装饰器 ≈ 注解 + AOP。但 Python 装饰器是运行时的函数变换，比 Java 注解更灵活。

**常用内置装饰器**：`@property`、`@classmethod`、`@staticmethod`、`@functools.lru_cache`、`@functools.wraps`

## 2. 生成器

```python
# 生成器函数（用 yield 代替 return）
def fibonacci():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# 惰性求值，不占内存
fib = fibonacci()
next(fib)   # 0
next(fib)   # 1
next(fib)   # 1
next(fib)   # 2

# 生成器表达式（类似推导式，但用圆括号）
squares = (x**2 for x in range(1000000))  # 不占内存
sum(squares)  # 用到时才计算

# yield from（委托给子生成器）
def chain(*iterables):
    for it in iterables:
        yield from it

list(chain([1, 2], [3, 4]))  # [1, 2, 3, 4]
```

**Java 类比**：生成器 ≈ Java 的 `Stream`（惰性求值），但语法更简洁。`yield` ≈ Java 没有直接对应（需要手写 Iterator）。

**为什么重要**：处理大数据时，生成器不把所有数据加载到内存。读 10GB 日志文件，生成器只占几 KB 内存。

## 3. 上下文管理器

```python
# 基本用法
with open("file.txt", "r") as f:
    content = f.read()
# 自动关闭文件，即使发生异常

# 自定义上下文管理器
class Timer:
    def __enter__(self):
        self.start = time.time()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.elapsed = time.time() - self.start
        print(f"Elapsed: {self.elapsed:.2f}s")
        return False  # 不吞掉异常

with Timer():
    time.sleep(1)  # 打印: Elapsed: 1.00s

# 更简洁的写法：contextlib
from contextlib import contextmanager

@contextmanager
def timer():
    start = time.time()
    yield
    print(f"Elapsed: {time.time() - start:.2f}s")
```

**Java 类比**：`with` ≈ try-with-resources。`__enter__`/`__exit__` ≈ `AutoCloseable.close()`。

## 4. 异常处理

```python
# 基本结构
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Error: {e}")
except (ValueError, TypeError) as e:
    print(f"Other error: {e}")
else:
    print("No error occurred")  # 没有异常时执行
finally:
    print("Always executes")    # 总是执行

# 自定义异常
class AppError(Exception):
    def __init__(self, message: str, code: int):
        super().__init__(message)
        self.code = code

raise AppError("Not found", 404)

# 异常链
try:
    open("missing.txt")
except FileNotFoundError as e:
    raise AppError("File missing", 404) from e
```

**注意**：Python 用 `except Exception` 而不是 `catch`。不要用裸 `except:`（会捕获 KeyboardInterrupt 等系统信号）。

## 5. 类型标注

```python
from typing import Optional, Union, Callable, TypeVar, Generic

# 基本类型
def greet(name: str) -> str:
    return f"Hello, {name}"

# 可选类型
def find_user(id: int) -> Optional[str]:  # 返回 str 或 None
    ...

# 联合类型（Python 3.10+ 用 |）
def process(value: int | str) -> int | str:
    ...

# 泛型
T = TypeVar("T")

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

# Callable 类型
def apply(func: Callable[[int], int], value: int) -> int:
    return func(value)
```

**工具**：用 `mypy` 做静态类型检查（`pip install mypy && mypy src/`）。

**Java 类比**：Python 类型标注是可选的，运行时不强制。Java 类型是强制的。但大型项目推荐加类型标注，IDE 支持更好。

## 6. 并发编程

```python
# asyncio（异步 I/O，最常用）
import asyncio

async def fetch_data(url: str) -> str:
    print(f"Fetching {url}...")
    await asyncio.sleep(1)  # 模拟 I/O
    return f"Data from {url}"

async def main():
    # 并发执行
    results = await asyncio.gather(
        fetch_data("url1"),
        fetch_data("url2"),
        fetch_data("url3"),
    )
    print(results)

asyncio.run(main())

# 线程（适合 I/O 密集）
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=4) as executor:
    futures = [executor.submit(fetch, url) for url in urls]
    results = [f.result() for f in futures]

# 进程（适合 CPU 密集）
from concurrent.futures import ProcessPoolExecutor

with ProcessPoolExecutor() as executor:
    results = list(executor.map(cpu_heavy_function, data))
```

**Java 类比**：
- `asyncio` ≈ Java 的 `CompletableFuture` + 事件循环
- `ThreadPoolExecutor` ≈ Java 的 `ExecutorService`
- `ProcessPoolExecutor` ≈ Java 的 `ForkJoinPool`（但真正多进程）

**关键区别**：Python 有 GIL（全局解释器锁），多线程不能并行执行 CPU 密集任务。CPU 密集用多进程，I/O 密集用异步或多线程。

## 7. 测试

```python
# pytest（推荐，比 unittest 简洁）
# 文件: test_math_utils.py

import pytest

def add(a: int, b: int) -> int:
    return a + b

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

def test_add_type_error():
    with pytest.raises(TypeError):
        add("1", 2)

# fixture（测试前置/后置）
@pytest.fixture
def sample_data():
    return {"users": ["Alice", "Bob"]}

def test_user_count(sample_data):
    assert len(sample_data["users"]) == 2

# 参数化测试
@pytest.mark.parametrize("a,b,expected", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
])
def test_add_parametrize(a, b, expected):
    assert add(a, b) == expected
```

```bash
# 运行测试
pytest                    # 自动发现并运行所有测试
pytest -v                 # 详细输出
pytest --cov=src          # 带覆盖率
```

**Java 类比**：`pytest` ≈ JUnit 5，但更简洁。`fixture` ≈ `@BeforeEach`。`@pytest.mark.parametrize` ≈ `@ParameterizedTest`。

## 8. 常用设计模式

```python
# 单例（用模块级变量，Python 最自然的方式）
# config.py
class _Config:
    def __init__(self):
        self.debug = False

config = _Config()  # 模块导入时只创建一次

# 观察者模式
class EventEmitter:
    def __init__(self):
        self._listeners: dict[str, list[Callable]] = {}

    def on(self, event: str, callback: Callable):
        self._listeners.setdefault(event, []).append(callback)

    def emit(self, event: str, *args, **kwargs):
        for callback in self._listeners.get(event, []):
            callback(*args, **kwargs)

# 策略模式（函数是一等公民，不需要接口）
def sort_by_name(user): return user["name"]
def sort_by_age(user): return user["age"]

users.sort(key=sort_by_name)  # 切换策略只需换函数
```

**Python 特色**：很多 Java 设计模式在 Python 中不需要，因为函数是一等公民、动态类型、鸭子类型。
