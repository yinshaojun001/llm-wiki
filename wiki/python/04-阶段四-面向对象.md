# 阶段四：面向对象编程（第 7-8 周）

> 目标：掌握 Python 的 OOP，理解与 Java OOP 的核心差异

---

## 1. 类基础

```python
class User:
    # 类变量（所有实例共享，类似 Java static）
    species = "human"

    def __init__(self, name: str, age: int):
        # 实例变量（每个实例独立）
        self.name = name
        self.age = age

    # 实例方法（第一个参数必须是 self）
    def greet(self) -> str:
        return f"Hi, I'm {self.name}"

    # 类方法（类似 Java static method，但能访问类）
    @classmethod
    def from_string(cls, s: str) -> "User":
        name, age = s.split(",")
        return cls(name.strip(), int(age.strip()))

    # 静态方法（和类无关，只是放在类里）
    @staticmethod
    def is_adult(age: int) -> bool:
        return age >= 18

# 使用
user = User("Alice", 30)
user.greet()                        # "Hi, I'm Alice"
user2 = User.from_string("Bob, 25") # 工厂方法
User.is_adult(20)                   # True
```

**Java 类比**：
- `self` ≈ `this`（但 Python 必须显式写）
- `__init__` ≈ 构造器
- `@classmethod` ≈ `static` 工厂方法
- `@staticmethod` ≈ `static` 方法

## 2. 访问控制

```python
class Account:
    def __init__(self, balance: float):
        self._balance = balance    # 约定"受保护"（单下划线）
        self.__secret = "hidden"   # 名称改写（双下划线）

    def get_balance(self):
        return self._balance

acc = Account(100)
acc._balance       # 100（能访问，但不该这么做）
acc.__secret       # AttributeError!
acc._Account__secret  # "hidden"（可以绕过，但别这么做）
```

**Python 哲学**：没有真正的 private。"We're all consenting adults here." 单下划线是约定，不是强制。

**Java 类比**：Python 的 `_` ≈ `protected`，`__` ≈ 一种弱化的 `private`。但 Python 更信任开发者。

## 3. 继承

```python
class Animal:
    def __init__(self, name: str):
        self.name = name

    def speak(self) -> str:
        raise NotImplementedError("子类必须实现")

class Dog(Animal):
    def speak(self) -> str:
        return f"{self.name}: Woof!"

class Cat(Animal):
    def speak(self) -> str:
        return f"{self.name}: Meow!"

# 多态
animals = [Dog("Rex"), Cat("Tom")]
for animal in animals:
    print(animal.speak())
```

### 多重继承

```python
class Flyable:
    def fly(self):
        return "flying"

class Swimmable:
    def swim(self):
        return "swimming"

class Duck(Animal, Flyable, Swimmable):
    pass

duck = Duck("Donald")
duck.fly()    # "flying"
duck.swim()   # "swimming"
```

**MRO（方法解析顺序）**：Python 用 C3 线性化算法决定调用顺序。用 `ClassName.__mro__` 查看。

**Java 类比**：Java 只能单继承 + 多实现接口。Python 支持多继承，但容易混乱，慎用。

## 4. 魔术方法（Dunder Methods）

```python
class Vector:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    # 字符串表示
    def __repr__(self) -> str:
        return f"Vector({self.x}, {self.y})"

    def __str__(self) -> str:
        return f"({self.x}, {self.y})"

    # 运算符重载
    def __add__(self, other: "Vector") -> "Vector":
        return Vector(self.x + other.x, self.y + other.y)

    def __mul__(self, scalar: float) -> "Vector":
        return Vector(self.x * scalar, self.y * scalar)

    # 比较
    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Vector):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    # 容器协议
    def __len__(self) -> int:
        return 2

    def __getitem__(self, index: int) -> float:
        if index == 0: return self.x
        if index == 1: return self.y
        raise IndexError

v1 = Vector(1, 2)
v2 = Vector(3, 4)
v1 + v2         # Vector(4, 6)
v1 * 3          # Vector(3, 6)
v1 == v2        # False
len(v1)         # 2
v1[0]           # 1
```

**常用魔术方法**：

| 方法 | 触发条件 | Java 对应 |
|---|---|---|
| `__init__` | 构造 | 构造器 |
| `__repr__` | `repr(obj)` | `toString()`（调试用） |
| `__str__` | `str(obj)`, `print()` | `toString()`（展示用） |
| `__add__` | `a + b` | 无（不可重载运算符） |
| `__eq__` | `a == b` | `equals()` |
| `__lt__` | `a < b` | `Comparable.compareTo()` |
| `__len__` | `len(obj)` | `.size()` |
| `__getitem__` | `obj[key]` | `.get(index)` |
| `__contains__` | `x in obj` | `.contains()` |
| `__iter__` | `for x in obj` | `Iterable.iterator()` |

## 5. 属性（Property）

```python
class Temperature:
    def __init__(self, celsius: float):
        self._celsius = celsius

    @property
    def celsius(self) -> float:
        return self._celsius

    @celsius.setter
    def celsius(self, value: float):
        if value < -273.15:
            raise ValueError("低于绝对零度")
        self._celsius = value

    @property
    def fahrenheit(self) -> float:
        return self._celsius * 9/5 + 32

t = Temperature(100)
t.celsius        # 100（像属性一样访问）
t.celsius = 50   # 调用 setter
t.fahrenheit     # 122.0（只读属性）
```

**Java 类比**：`@property` ≈ Java 的 getter/setter，但语法更自然。Python 不需要 `getCelsius()` / `setCelsius()`。

## 6. 数据类（dataclass）

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    label: str = "origin"  # 默认值

# 自动生成 __init__, __repr__, __eq__
p = Point(3.0, 4.0)
print(p)          # Point(x=3.0, y=4.0, label='origin')
p == Point(3.0, 4.0)  # True
```

**Java 类比**：`@dataclass` ≈ Java 的 `record`。自动生成样板代码。

```python
# 更多选项
@dataclass(frozen=True)   # 不可变，类似 record
class Config:
    host: str
    port: int = 8080

@dataclass(order=True)    # 自动生成比较方法
class Student:
    name: str
    grade: float
```

## 7. 抽象类

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self) -> float:
        ...

    @abstractmethod
    def perimeter(self) -> float:
        ...

class Circle(Shape):
    def __init__(self, radius: float):
        self.radius = radius

    def area(self) -> float:
        import math
        return math.pi * self.radius ** 2

    def perimeter(self) -> float:
        import math
        return 2 * math.pi * self.radius

# Shape()  # TypeError! 不能实例化抽象类
```

**Java 类比**：`ABC` + `@abstractmethod` ≈ `abstract class` + `abstract method`。
