# 信任基础设施

> AI 工具的竞争，正在从能力竞争升级为信任基础设施竞争。

## 定义

当 AI Agent 从"辅助工具"走向"任务执行系统"，用户关心的问题从"能不能做"变成"敢不敢长期用"。信任基础设施是支撑这个"敢"字的一整套能力。

## 六大组成部分

| 能力 | 具体表现 |
|---|---|
| **计费透明度** | 用量仪表盘、消耗预测、异常提醒 |
| **权限边界控制** | 自动执行的授权范围、越权阻断 |
| **异常回滚** | 出错后能追踪、回退、恢复 |
| **状态恢复** | 长会话、跨会话状态持续可信 |
| **配置标准化** | 工作流配置可迁移（如 [[concepts/agents-md]]）|
| **跨平台兼容** | macOS/Windows/Linux 体验一致 |

## 为什么是护城河

能力强的工具容易被追上，但信任体系一旦建立，切换成本极高：

- 用户的工作流、权限配置、历史状态都绑定在这套体系里
- 企业采购看的正是"可解释性、可观测性、风险管理"

## 典型案例

- **正面**：[[entities/claude-code]] AGENTS.md 标准化配置
- **负面**：Claude Code Auto Mode 执行 `git reset --hard` bug（2026-03-28）、幽灵限流危机（2026-03-29）
