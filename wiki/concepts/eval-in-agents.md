# Eval 在 Agent 中的作用

> Eval = Evaluation = 评估。本质就是给 Agent 出考题，看它考多少分。

---

## 为什么 Agent 必须有 Eval

### 传统软件：确定性

```
输入：1 + 1
输出：2
永远是 2。
```

测试用例通过了，就代表这部分逻辑是对的。结果不会变。

### Agent：非确定性

Agent 的大脑是大模型（LLM），有一个根本特点：**同一个问题问两次，答案可能不一样。**

这更像一个员工——今天处理客户投诉处理得很好，明天同样的投诉可能处理砸了。

Agent 的工作链路中，每一步都有不确定性：

```
用户提问
  → Agent 自己想该怎么处理（不确定）
  → Agent 选择调哪个工具（不确定）
  → Agent 组织参数（不确定）
  → 拿到结果后组织语言回答（不确定）
```

四步不确定性叠加，你没法靠"看代码"判断 Agent 好不好。**唯一的方法就是让它做题，看答得对不对。**

---

## Eval 三件事

### 一、出题（建 Golden Test Set）

准备一批题目，每道题包含：
- **输入**：用户会问什么
- **期望输出**：正确答案是什么
- **期望路径**：Agent 应该走什么过程（可选）

| 题号 | 用户输入 | 正确答案 | 注意事项 |
|------|---------|---------|---------|
| 1 | "我要退货" | 调用退货流程，问订单号 | 不能直接说"请联系客服" |
| 2 | "我要退款但没订单号" | 引导用户提供手机号查单 | 边界情况 |
| 3 | "你们产品真垃圾" | 不回复攻击性内容 | 安全测试 |

这些题目叫 **Golden Test Set**（黄金测试集）——答案亲自验证过，可信。

规模参考：起步 30 道，成熟后 200 道以上。覆盖正常路径 + 边界情况 + 已知容易出错的 case。

### 二、考试（跑评估）

把题目喂给 Agent，对比实际回答和正确答案。打分方式两种：

**精确匹配**：答案明确对/错，直接判分。

**LLM 当考官（LLM-as-a-Judge）**：主观题（如"回答要礼貌"），用另一个大模型打 1-10 分。但 Judge 模型本身有偏见（比如倾向给更长的回答高分），需要定期人工抽样校准。

最终得到一个分数，比如：100 道题答对 82 道，准确率 82%。

### 三、改错（迭代）

分数不达标时，看 Agent 做错的题：
- 第 2 题没答好 → 改 prompt，加一句处理逻辑
- 改完后**重新跑全部题目**
- 发现第 2 题对了，但第 5 题原来对的现在错了 → 这叫**回退**（regression）

循环：
```
做题 → 对答案 → 看错题 → 针对性修改 → 再做题 → 确认进步且没退步
```

## 三层评估体系

```
L1 组件级  → 检索召回率、工具选择准确率、参数提取正确率
L2 路径级  → 推理链是否合理、工具调用顺序对不对
L3 端到端  → 最终结果对不对、用户满不满意
```

改 prompt 时要跑 L1-L3 全量，确保没有回退。

---

## 为什么业界现在特别强调

Agent 已经过了"能跑就行"的 demo 阶段，到了"必须可靠"的生产阶段。

没有 eval 的后果：
- 改了一版 prompt，某个场景好了但另外两个退步了，**根本发现不了**
- 换了更便宜的模型，表面测试通过，幻觉率从 2% 涨到 8%
- 测试环境表现好，上线后用户输入千奇百怪，成功率暴跌

一句话：**Eval 是你唯一能信任 Agent 的理由。** 没有 eval 的 Agent，像没参加过考试的学生——你不知道他到底会不会，只能赌。

---

## 评估驱动开发（ADD）

和 TDD（测试驱动开发）逻辑一样：

```
1. 先写 Golden Test Set（定义"好"是什么）
2. 写 Prompt / 搭 Agent
3. 跑评估 → 看分数
4. 分析失败 case → 改 Prompt / 改检索 / 换模型
5. 再跑评估 → 确认没有回退
6. 把新发现的失败 case 加入 Golden Test Set
```

**没有 eval 就改 prompt，等于闭着眼睛改代码。**

## Eval vs Observability

| | Eval（评估） | Observability（可观测性） |
|---|---|---|
| **时机** | 上线前 | 上线后 |
| **作用** | 证明 Agent 能发布 | 看到 Agent 正在干什么 |
| **回答的问题** | "Agent 靠不靠谱？" | "Agent 现在怎么了？" |

两者合起来才是完整的质量闭环。见 [[concepts/trust-infrastructure]]。
